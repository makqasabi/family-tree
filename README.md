# مسودة شجرة عائلة عثمان بن عبدالله بن عبدالعزيز القصبي وأرحامهم وأصهارهم

> *(بانتظار التعديلات والإضافات)*

موقع خاص لعرض وتحرير شجرة العائلة بالعربية، مع دعم النسب الأبوي والأمومي وعلاقة الرضاعة. الموقع ثابت بالكامل (HTML/CSS/JS) — البيانات تُخزَّن مشفّرة AES-GCM ويُشتق المفتاح من كلمة المرور عبر PBKDF2.

## التشغيل المحلي

المتصفح يحتاج خادمًا محليًا لقراءة `data/seed.json` (لا يعمل من `file://`):

```powershell
# من داخل المجلّد:
python -m http.server 8000
# ثم افتح:  http://localhost:8000
```

أو إن كان لديك Node:

```powershell
npx http-server -p 8000
```

## الاستخدام لأول مرة

1. افتح `http://localhost:8000` في المتصفح.
2. اختر "ابدأ من البيانات الأولية (seed.json)" — تظهر لوحة تعيين كلمة المرور.
3. أدخل كلمة مرور قوية وتلميحًا اختياريًا. سيتم تشفير `seed.json` وحفظ النسخة المشفّرة في `localStorage` للمتصفح.
4. اضغط "تشفير وبدء" — وستفتح الشجرة.

⚠️ **لا يمكن استرجاع كلمة المرور.** لا توجد آلية لاستعادتها — احتفظ بها في مدير كلمات مرور.

## النسخ الاحتياطي والمشاركة

- **زر التصدير ⇩** ينزّل ملف `family.enc.json` (مشفّر بكلمة المرور الحالية). احتفظ بهذا الملف في مكان آمن أو ضعه في `data/family.enc.json` لتُحمَّل تلقائيًا عند فتح الموقع من جهاز جديد.
- **زر الاستيراد ⇧** يقرأ ملف `family.enc.json` ويستبدل النسخة المحلية، ثم يطلب كلمة المرور.
- **زر القفل 🔒** يفرغ كلمة المرور من الذاكرة ويعيد طلبها.

## النشر مع كلمة المرور

لمشاركة الموقع مع أفراد العائلة:

1. أنشئ ملف `data/family.enc.json` (عبر زر التصدير).
2. **احذف `data/seed.json`** من النشر (يحتوي على البيانات بدون تشفير).
3. ارفع المحتوى إلى GitHub Pages أو Netlify أو أي استضافة ساكنة.
4. شارك الرابط مع أفراد العائلة + كلمة المرور عبر قناة آمنة.

أي شخص يفتح الرابط سيرى نموذج إدخال كلمة المرور؛ بدونها لا تُفك الشِفرة.

## بنية البيانات

البيانات في `data/seed.json` بصيغة JSON — يمكنك تحريرها يدويًا قبل التشفير الأول. الحقول الأساسية لكل شخص:

```json
{
  "fullName": "عثمان عبدالله عبدالعزيز القصبي",
  "given": "عثمان",
  "family": "القصبي",
  "gender": "male",
  "fatherId": "p_abdullah_abdulaziz_qassabi",
  "motherId": "p_moudi_alhamdan",
  "birth": { "year": 1898, "approximate": true, "place": "الزبير" },
  "death": { "year": 1974 },
  "places": ["الزبير", "البحرين", "دارين", "الرياض"],
  "occupations": ["خبير صنف في الحنطة"],
  "tribe": "...",
  "isPlaceholder": false,
  "external": false,
  "notes": "..."
}
```

- **`isPlaceholder: true`** — الشخص مذكور للنَسَب فقط ولا توجد تفاصيل موثّقة.
- **`external: true`** — الشخص من عائلة الزوج/الزوجة (نسب آخر).
- **الزواج** يُسجَّل في مصفوفة `marriages` كل عنصر `{ id, spouseIds: [a, b], order? }`.
- **الرضاعة** في مصفوفة `milkRelations` بنوع `milk-mother` أو `milk-sibling`.
- **`rootIds`** — قائمة بمعرّفات أصول الشجرة (تُعرض في القمة). يكتشف النظام تلقائيًا أي شخص بدون أب لديه أبناء ويعرضه كجذر إضافي.

## الميزات

- **عرضان:** قائمة هرمية قابلة للطي (مناسبة للأسماء العربية الطويلة)، وشجرة بصرية بـ D3.js مع تكبير وتحريك. كلاهما من اليمين إلى اليسار.
- **بحث فوري** بالاسم — مع تجاهل الفروقات الإملائية (إ/أ/آ/ا، ى/ي، ة/ه).
- **بطاقة الشخص:** اسم كامل، نسب موصول، الميلاد/الوفاة، الأماكن، المهن، الأبوان، الزواج (مع ترتيب الأزواج)، الإخوة، الأبناء، الرضاعة.
- **إضافة شخص:** نموذج بسيط مع منتقي للأب/الأم/الزوج. تُحفظ التغييرات تلقائيًا (مشفّرة) في `localStorage`.
- **تعديل/حذف:** من بطاقة الشخص. الحذف يفك ارتباط الأبناء وأزواج المحذوف.
- **تعيين "هذا أنا":** يميّز الشخص الحالي بحدّ سميك في الشجرة البصرية.

## الخصوصية

- التشفير AES-GCM 256-bit. اشتقاق المفتاح PBKDF2 مع 250,000 تكرار وSHA-256.
- البيانات لا تُرسَل لأي خادم — كل العمل في المتصفح.
- التلميح (`hint`) **غير مشفّر** ويُحفظ في `localStorage`؛ لا تكتب فيه شيئًا يكشف كلمة المرور.
- ⚠️ ملف `seed.json` نصّ صريح — احذفه قبل النشر العام.

## التحقّق من سلامة بيانات seed.json

ملف `data/seed.json` يُتحقّق منه بسكربت Node:

```powershell
node tools/validate-seed.js
```

يُجري الفحوصات التالية ويُخرج تقريرًا مصنّفًا (ERROR / WARN / INFO):

**الأخطاء (يجب إصلاحها — تُسبب الخروج بكود ١):**
- `missing-fullname` — شخص بدون `fullName`
- `name-given-mismatch` — `given` لا يظهر داخل `fullName`
- `self-father` / `self-mother` — `fatherId` أو `motherId` يساوي معرّف الشخص نفسه
- `missing-father` / `missing-mother` — مرجع لشخص غير موجود
- `father-not-male` / `mother-not-female` — جنس الوالد لا يطابق
- `missing-tribal-root` — `tribalRootId` لشخص غير موجود
- `birth-after-death` — سنة الميلاد بعد سنة الوفاة
- `father-not-older` / `mother-not-older` — والد ولد بعد ابنه
- `marriage-no-id` / `marriage-bad-spouses` / `marriage-self` / `marriage-missing-spouse` / `marriage-duplicate`
- `milk-no-id` / `milk-missing-person`
- `missing-root` / `missing-current-user` / `missing-admin`
- `ancestry-cycle` / `ancestry-cycle-maternal` — حلقة في النسب

**التحذيرات (مستحبّ إصلاحها):**
- `name-family-mismatch` — اسم العائلة لا يظهر في `fullName`
- `id-given-mismatch` — معرّف الشخص لا يحتوي على ترانسليتيريشن للاسم الأول (مثلًا `p_anas_X` لشخص اسمه "علي")
- `birth-year-odd` / `death-year-odd` — السنة خارج النطاق ١٥٠٠–٢١٠٠
- `bad-month` / `bad-day` — قيمة شهر/يوم غير منطقية
- `marriage-year-odd` — سنة الزواج خارج النطاق
- `marriage-same-gender` — كلا الزوجين بنفس الجنس

**ملاحظات معلوماتية:**
- `duplicate-fullname` — أكثر من سجل بنفس `fullName` المُطبَّع
- `orphan-placeholder` — placeholder بدون أي مرجع وارد (متروك للتوسعة لاحقًا)

سيتم تشغيل هذا السكربت تلقائيًا قبل أي تحديث جوهري لـ `seed.json` لمنع تسرّب أخطاء بنيوية إلى المستخدمين.

## ملاحظات وتفاصيل لاحقة

- بيانات النسب لزوجات/أزواج الأقارب موسومة `isPlaceholder: true` — لتوسيعها لاحقًا أضف الأبوين عبر زر "إضافة شخص" واربطهم.
- علاقة الرضاعة الحالية لعثمان (١٨٩٨م) — أخت من عائلة العويدي، بدون اسم كامل. يمكن تحديثها لاحقًا.
- بعض التواريخ تقريبية (`approximate: true` مع `approxRange`).
- النسب: استُخدم `fullName` مع البنية العربية الطويلة (الاسم + الأب + الجد + ... + اسم العائلة)؛ والتسلسل الكامل يُستخرَج آليًا في بطاقة الشخص.
